Hydra-Swarm architecture v0.7.0

Deterministic gates decide; probabilistic tools inform. A lead plans and judges, four vendor heads implement in isolated worktrees, and nothing a worker claims is real until it crosses the trust boundary.

Claude Codex Kimi OpenCode/GLM trust boundary human-authorized

Head detection (v0.7.0)

  • detect-heads probes all four vendor CLIs, enumerates opencode's configured models and active model, and checks srt for Kimi write-capability.
  • Snapshot: ~/.local/state/hydra/heads.json, refreshed by run-init (ledger event heads_detected).
  • allocate never recommends an uninstalled head (degrades with a warning when nothing probes available); dispatch fails fast naming available substitutes — never auto-substitutes.

Worker environment (v0.7.0)

  • HYDRA_NODE_BIN: a verified node ≥ 22.6 exported into every worker env; the worker prompt names the one-line PATH fix.
  • Kimi preflight (worker-devenv): toolchain, per-task store/cache dirs (canonicalized tmp base — srt matches physical paths), manifest-derived network domains.

Runtime

  • Default: Bun-compiled single binary (npm run build:bin); source lane falls back automatically.
  • hydra version reports the version the binary was built from; hydra help lists every subcommand with its arguments.
